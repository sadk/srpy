Simple Remote Python (SRPy) intends to harness the power of the Python by allowing multiple python programs (instances) to seamlessly communicate and share information between each other. As such, each CPU (or core) in each computer can be considered as an individual that can request for other individuals to store information or run processing tasks. In this context, SRPy abstracts itself from the physical boundaries between processors and computers by looking at them simply as volunteer workers that are able to deal with workloads - this workers can even work in a social manner by communicating with each other on a non-centralized way, sharing information and processing requests.

Quick Start - Starting a Server:

  • Create a server on a computer by running the srpyapp.py script.

Example:

toseinin:srpy paxcal$ python2.5 srpy/srpyapp.py

SRPy Server - Simple Remote Python, Network Server

http://code.google.com/p/srpy/ - updates, documentation, examples and support

URI info:

PYRO://192.168.1.23:7829/c0a801171909517cdef4645b78192636

Write 'quit' or 'exit' to exit...

  • Copy the URI info as it will be needed when connecting with a client.

Note: run the server with '--help' to see a list of available options, srpyapp.py is able to generate multiple servers in one go, optimally you should have one server per CPU available.

Quick Start - Connecting to a Server:

Here's an example of connecting to the created server:

>>> import srpy
>>> pyeng=srpy.PythonEngine("PYRO://192.168.1.23:7829/c0a801171909517cdef4645b78192636")

Check the PythonEngine and EngBox classes for more info on how to work with remote python engines.
