27  """ 
28  Simple Remote Python (SRPy) intends to harness the power of the Python by 
29  allowing multiple python programs (instances) to seamlessly communicate and 
30  share information between each other. As such, each CPU (or core) in each 
31  computer can be considered as an individual that can request for other 
32  individuals to store information or run processing tasks. In this context, 
33  SRPy abstracts itself from the physical boundaries between processors and 
34  computers by looking at them simply as volunteer workers that are able to 
35  deal with workloads - this workers can even work in a social manner by 
36  communicating with each other on a non-centralized way, sharing information 
37  and processing requests. 
38   
39  B{Quick Start} - X{Starting a Server}: 
40   
41      - Create a server on a computer by running the srpyapp.py script. 
42       
43  Example: 
44   
45  C{toseinin:srpy paxcal$ python2.5 srpy/srpyapp.py} 
46   
47  C{SRPy Server - Simple Remote Python, Network Server} 
48   
49  C{http://code.google.com/p/srpy/ - updates, documentation, examples and support} 
50   
51  C{Starting Basic Server...} 
52   
53  C{URI info:} 
54   
55  C{PYRO://192.168.1.23:7829/c0a801171909517cdef4645b78192636} 
56   
57  C{Write 'quit' or 'exit' to exit...} 
58   
59      - Copy the X{URI info} as it will be needed when connecting with a client.  
60   
61  I{Note:} run the server with '--help' to see a list of available options, 
62  srpyapp.py is able to generate multiple servers in one go, optimally you 
63  should have one server per CPU available. 
64   
65   
66  B{Quick Start} - X{Connecting to a Server}: 
67   
68  Here's an example of connecting to the created server: 
69   
70  >>> import srpy 
71  >>> pyeng=srpy.PythonEngine("PYRO://192.168.1.23:7829/c0a801171909517cdef4645b78192636") 
72   
73  Check the L{PythonEngine <srpyclient.PythonEngine>} and 
74  L{EngBox <srpyclient.PythonEngine>} classes for more info on how to work with 
75  remote python engines. 
76   
77  """
